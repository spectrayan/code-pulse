# Design Patterns

Common GoF and architectural patterns for robust, maintainable, and reusable object-oriented design.

## Creational Patterns
Use patterns like Factory Method, Abstract Factory, Builder, and Singleton to decouple object creation from its use.

## Structural Patterns
Use Adapter, Bridge, Composite, Decorator, Facade, Flyweight, and Proxy to organize objects and classes into larger structures.

## Behavioral Patterns
Use Command, Interpreter, Iterator, Mediator, Memento, Observer, State, Strategy, Template Method, and Visitor to define algorithms and manage communication between objects.

## Favor Composition over Inheritance
Prefer object composition (using objects of other classes) over class inheritance to achieve flexibility and code reuse.

## Single Responsibility Pattern (SRP)
Every class should have only one reason to change, meaning it should perform a single, well-defined task.

## Open/Closed Principle (OCP)
Software entities (classes, modules, functions) should be open for extension but closed for modification.
