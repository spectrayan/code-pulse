# Clean Code Principles

Key principles from Robert C. Martin's Clean Code methodology.

## Meaningful Names
Use intention-revealing names. Avoid disinformation, encodings, and mental mapping.

## Small Functions
Functions should be small, do one thing, and operate at a single level of abstraction.

## Comments
Good code is self-documenting. Use comments only to explain *why*, not *what*. Avoid redundant or misleading comments.

## Error Handling
Use exceptions rather than error codes. Don't return or pass null. Provide context with exceptions.

## DRY (Don't Repeat Yourself)
Every piece of knowledge should have a single, unambiguous representation in the system.

## Code Smells
Watch for: long methods, large classes, feature envy, data clumps, primitive obsession, and shotgun surgery.
